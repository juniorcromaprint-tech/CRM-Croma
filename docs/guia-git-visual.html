📖 Glossário — Os 10 Termos Essenciais
🗄️
Git
Sistema de controle de versão
Registra cada mudança feita no código. Você pode voltar a qualquer ponto no tempo, ver quem fez o quê, e trabalhar em paralelo sem conflito.
📎 Analogia: É como o histórico de versões do Google Docs, mas para código inteiro.
💾
Commit
git commit -m "mensagem"
Um ponto de salvamento com descrição. Cada commit grava exatamente o que mudou e por quê. Você pode ter centenas deles.
📎 Analogia: É como o Ctrl+Z com nome — "adicionei a página de expedição".
🌿
Branch
git branch nome-do-branch
Uma cópia paralela do código para trabalhar sem risco. Você faz as mudanças no galho (branch) sem afetar o tronco (main).
📎 Analogia: É como uma mesa de trabalho separada — você monta o projeto ali e só leva pro salão quando está pronto.
🏠
Main
branch principal
O branch oficial. É o código que está no ar, rodando em crm-croma.vercel.app. Tudo que entra aqui vai pro site.
📎 Analogia: É a vitrine da loja — só vai pra lá o que está pronto e aprovado.
☁️
Push
git push
Enviar seus commits do computador para o GitHub (a nuvem). Sem push, as mudanças ficam só na sua máquina.
📎 Analogia: Commit = salvar o arquivo. Push = fazer upload pro Google Drive.
⬇️
Pull
git pull
Baixar as mudanças do GitHub para o computador. É o inverso do push — traz o que outras pessoas (ou outras sessões) fizeram.
📎 Analogia: É como sincronizar o Dropbox — puxa o que está na nuvem.
🔀
Merge
git merge branch
Juntar um branch de trabalho de volta no main. É quando o galho volta pro tronco — as mudanças passam a valer oficialmente.
📎 Analogia: É como juntar duas planilhas numa só — combina o trabalho.
📋
Pull Request (PR)
GitHub → New Pull Request
Um pedido de revisão antes do merge. O GitHub mostra todas as mudanças em detalhes para você aprovar ou pedir ajustes antes de ir pro ar.
📎 Analogia: É como enviar uma prova pra revisão antes de imprimir — você confere e aprova.
🚀
Deploy
Automático pela Vercel
Publicar o código no servidor. No seu caso a Vercel faz isso sozinha: cada merge no main → site atualizado em ~1 minuto.
📎 Analogia: É como o "Publicar" do Instagram — o conteúdo vai pro ar.
📂
Worktree
git worktree add
Uma pasta isolada para trabalhar em outro branch sem mexer na pasta principal. Duas cópias do projeto rodando lado a lado.
📎 Analogia: É como ter duas mesas de trabalho — enquanto uma versão está na produção, você monta outra na mesa ao lado.

💡 Dicas Importantes
✅
Tudo automático pra você
O Claude (eu) cuida de todo o fluxo Git — commits, branches, push e PRs. Você só precisa aprovar o PR no GitHub quando estiver satisfeito.
🔒
Main é protegido
Nunca trabalho direto no main. Sempre crio um branch separado, trabalho lá, e só junto quando tudo está testado e aprovado. Zero risco pro site no ar.
⏱️
Deploy em 1 minuto
Assim que um PR é mergeado no main, a Vercel detecta automaticamente e publica a nova versão. Sem precisar fazer nada — é tudo automático.
